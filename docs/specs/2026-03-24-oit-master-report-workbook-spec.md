# OIT Master Report Workbook Specification

## Summary

This document defines the expected structure, computations, formatting, and known limitations for the **primary-site OIT master workbook** generated by:

- `GET /api/report/templates/master.xlsx`

The canonical output filename for the master workbook is:

- `OIT_Master_Report_{YYYYMMDD}_{HHmm}.xlsx`

This specification is based on the current OIT reporting product requirements, with corrections for issues found in an earlier generated artifact. In particular:

- `First Response Time` degrades to `proxy` when Jira first-response elapsed data is missing.
- `Escalation Count` on `Trends` means tickets with a real escalation event on that day, not merely escalated tickets updated that day.
- Dashboard chart and sparkline source data lives on a hidden helper sheet, not visibly on the dashboard.
- Detail sheets use formula-driven total rows and formula-driven dashboard KPI links instead of hardcoded dashboard values.
- `Report Index` includes a leading status indicator column.
- `Data Gaps` includes any report that is effectively `proxy` for the current run, including `First Response Time` when elapsed data is partial.

## Workbook Architecture

The workbook contains 18 sheets arranged in four layers.

| Layer | Sheets | Purpose |
|---|---|---|
| Navigation | `Report Index` | Catalog of all metric sheets with metadata |
| Summary | `Executive Dashboard`, `Trends`, `Data Gaps` | Cross-metric summary, daily trends, known limitations |
| Metric Sheets | 14 total | One `7d` and one `30d` sheet for each included report template |

The seeded report families are:

| Report | Category | Baseline Readiness | Window Field | Grouping |
|---|---|---|---|---|
| Mean Time to Resolution | Executive | `ready` | `resolved` | `priority` |
| SLA Compliance Rate | Executive | `ready` | `created` | `sla_resolution_status` |
| Ticket Volume by Category | Executive | `ready` | `created` | `request_type` |
| Backlog Size & Aging | Operational | `ready` | `created` | aging bucket |
| Escalation Rate | Operational | `proxy` | `updated` | `assignee` |
| First Response Time | Operational | `ready` baseline, `proxy` when SLA elapsed data is missing | `created` | `sla_first_response_status` |
| First Contact Resolution | Quality | `proxy` | `resolved` | `request_type` |

## Time Windows

All windows are derived from the UTC generation date.

| Window | Label | Start | End |
|---|---|---|---|
| 7-day | `7 Day` | `end_date - 6 days` | `end_date` |
| 30-day | `30 Day` | `end_date - 29 days` | `end_date` |

`Δ Count vs Prior` is always:

- the current row count
- minus the count from the immediately preceding window of the same length

This is an integer delta, not a percentage.

## Sheet Schemas

### Common Metadata Block

Every metric sheet uses rows 1–11 in columns `A:B` for metadata:

1. `Report`
2. `Window`
3. `Window Field`
4. `Window Start`
5. `Window End`
6. `Category`
7. `Readiness`
8. `View`
9. `Description`
10. `Notes`
11. `Generated`

Row 12 is a spacer. Row 13 is the data-table header row.

Metadata rows are grouped/collapsed by default.

#### Escalation Rate

| Column | Header |
|---|---|
| A | Assignee |
| B | Count |
| C | Open |
| D | Avg TTR (h) |
| E | Median TTR (h) |
| F | P95 TTR (h) |
| G | P99 TTR (h) |
| H | Δ Count vs Prior |
| I | Escalation Rate % |

Unassigned values render as `(none)`.

Sort order: descending by `Count`.

Escalation is a proxy heuristic. A ticket is considered escalated if one or more of these occurred:

- assignee changed more than once
- priority increased
- escalation marker labels or components are present

For daily trends, `Escalation Count` means the number of tickets with an actual escalation event date on that day. It does not mean “all escalated tickets touched on that day.”

#### First Response Time

Same base columns as MTTR, with column `A = SLA Response`.

In the current workbook, the TTR-style columns on this sheet represent first-response elapsed hours when available. If Jira first-response elapsed time is missing for any relevant tickets, this report is treated as `proxy` for that run and appears in `Data Gaps`.

#### First Contact Resolution

Same base columns as MTTR, with column `A = Request Type`, plus:

| Column | Header |
|---|---|
| I | FCR % |

This remains a proxy metric.

### Executive Dashboard

Contains:

- KPI table for:
  - SLA Compliance Rate %

  - Backlog Count
  - Ticket Volume

- sparklines for the KPI series

Important implementation rule:

- dashboard helper data must not appear visibly on the dashboard grid
- chart and sparkline source ranges are stored on a hidden helper worksheet named like `Executive Dashboard Data`

## Metric Computation Rules

### TTR

For resolved-ticket metrics:

- `TTR = resolved_timestamp - created_timestamp`
- unit: hours

For backlog metrics:

- age is `generation_time - created_timestamp`
- unit: hours

### Percentiles

Percentiles are computed from raw ticket-level values, not grouped averages:

- median / P50
- P95
- P99

### SLA Compliance Rate

Use only `Met` and `BREACHED` in the rate denominator:

- `Met / (Met + BREACHED)`

`Running` and `(none)` appear in breakdowns but are excluded from the rate.

### Backlog

Backlog means tickets not in a terminal status.

### First Contact Resolution

Proxy logic uses resolved tickets and a one-touch style heuristic based on:

- first resolution timing
- low comment volume

### Escalation

Escalation remains a proxy. It uses changelog-derived reassignment and priority-increase events plus escalation markers. Daily escalation counts are based on recorded event dates, not generic last-updated dates.

## Formatting Rules

### Core Styling

- sheet title: bold, large, dark text
- metadata labels: bold
- metadata values: regular
- row 13 headers: bold white text on dark blue background
- data rows: standard readable font sizing

### Number Formats

- integer counts: `#,##0`
- hours: `0.0`
- rates/percentages: `0.0%`

### Conditional Formatting

- SLA-style KPI cells:
  - red when materially below threshold
  - amber when borderline
- SLA detail rows:
  - `BREACHED` rows shaded red
  - `Running` rows shaded amber
- backlog rows:
  - aging-bucket heat map from green through dark red
- MTTR rows:
  - P95/P99 thresholds use green / amber / red bands
- escalation and FCR sheets:
  - rate columns use green / amber / red bands
- proxy readiness cells:
  - yellow fill plus a warning comment on `B7`
- ready readiness cells:
  - green fill on `B7`

### Panes and Layout

- `Report Index`: freeze row 1
- `Trends`: freeze row 4
- metric sheets: freeze row 13
- detail sheets append a bold total row immediately after the data rows; SLA and First Response also append a second metric row for the met-rate KPI

## Edge Cases

- zero-count groups still render as zeroes, not `NaN` or blank error values
- missing SLA status becomes `(none)`
- unassigned assignee becomes `(none)`
- backlog buckets always render, even when empty
- if prior period has no data, `Δ Count vs Prior = current_count`
- if an entire window has no data, render the sheet structure with headers and zero/empty data rows as appropriate

## Validation Checklist

Before accepting a workbook, verify:

- all expected sheets are present
- `Report Index` row count matches the metric sheets
- readiness labels agree across `Report Index`, metadata, and `Data Gaps`
- `Executive Dashboard` shows no visible helper/source data leakage
- `Trends` has 30 daily rows
- detail sheets contain formula-based total rows and the expected extra SLA / First Response metric rows
- `Escalation Count` values are plausible and not driven by generic updates
- percentiles are present on the applicable sheets
- charts and sparklines exist
- number formats and conditional formatting are applied
- generation timestamp is internally consistent across sheets

## Known Limitations

- `Escalation Rate` is still a proxy until Jira has authoritative escalation markers or workflow transitions.
- `First Contact Resolution` is still a proxy until explicit contact-cycle data exists.
- `First Response Time` may degrade to `proxy` for a given run when Jira first-response elapsed values are missing.
